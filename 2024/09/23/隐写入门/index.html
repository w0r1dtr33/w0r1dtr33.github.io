扫盲

信息保护

  1. 数据加密
    • 通过加密秘钥进行对明文域的加密，加密成密文域，但是可能会导致乱码
  2. 图像隐写
    • 对正常图片进行嵌入，保证正常图片的情况下隐式写入密文，对观察者不可见

图像隐写方法

盲隐写和非盲隐写

  1. 盲隐写只需要加密后的图像和密钥(可选)
    • 实用
    • 隐蔽性
  2. 非盲隐写需要原始图像，通过比较来找到隐藏的信息
    • 更容易实现
    • 提高信息容量

空间域隐写

直接操作图像的像素信息来隐藏信息

  • 直接性：直接修改图片的像素颜色或者亮度之类的
  • 技术：常见的技术是最低有效位LSB方法，修改了像素值的后几位
  • 隐蔽性：由修改像素的数量决定
  • 鲁棒性：容易对图片造成损坏

频域隐写

在图像的变换域中隐藏信息，图像需要先被转换到另一个域(如傅里叶变换和小波变换)，然后在域中嵌入信息

  • 比空间域隐写实现更为复杂
  • 常见技术包括离散余弦变换和离散小波变换
  • 由于信息在变换域中是隐藏的，因此对原始图像的视觉影像更小

StegSolve

可以看到不同像素通道下的图片信息

首先先看像素通道有没有通过LSB隐藏的图片

  • File Format：文件格式及属性信息。
  • Data Extract：数据提取，主要用于提取比特并实现排列组合。
  • Stereogram Solver：立体图水平偏移控制，将图片水平偏移后再与原图叠加。
  • Frame Browser：帧浏览器，主要是对GIF之类的动图进行分解，可以查看每一帧的图像。
  • Image Combiner：拼图，图片拼接。

Data Extract是一个重要功能，左侧的“Bit Planes”是各通道以及可以选择的比特，“0”代表最低比特，“7”代表最高比特。右侧的“Order Settings”用于设置提取时的顺序，Exract By为按row（行）或column（列）提取像素；“Bit Order”中的MSB First指最高位是第一位，LSB First指最低位是第一位；“Bit Plane Order”用于设置RGB通道的顺序。选择好所有顺序后，单击“Preview”按钮，就能在窗口中预览提取的信息
